## 三、完整业务流程测试

通过 Gateway(:8080) 统一入口，模拟完整的用户购物流程：

```
步骤 1: POST /api/user/register     → 注册成功 (200)
步骤 2: POST /api/user/login        → 登录成功，获取 Session Cookie (200)
步骤 3: GET  /api/user/info         → 获取用户信息成功 (200)
步骤 4: GET  /api/product/list      → 浏览商品列表成功 (200)
步骤 5: GET  /api/product/1         → 查看商品详情成功 (200)
步骤 6: POST /api/cart/add          → 添加购物车成功 (200)
步骤 7: GET  /api/cart/list         → 查看购物车成功，含商品信息 (200)
步骤 8: POST /api/order/create      → 创建订单成功，库存扣减 (200)
步骤 9: GET  /api/order/list        → 查看订单成功，状态"待付款" (200)
步骤 10: POST /api/order/pay/{id}   → 模拟支付成功，状态"已付款" (200)
步骤 11: POST /api/user/logout      → 登出成功，Session 失效 (200)
步骤 12: GET  /api/user/info        → 登出后访问被拒 (401)
```

**全部 12 个步骤均通过测试。**

## 六、测试中发现并修复的问题

| # | 问题描述 | 影响范围 | 修复方案 |
|---|---------|---------|---------|
| 1 | Redis 无法序列化 `LocalDateTime` 类型 | 分类列表、轮播图查询报 500 | `RedisConfig` 添加 `JavaTimeModule` 支持 |
| 2 | 各服务 Session 存储在不同 Redis DB，Cookie 无法跨服务共享 | 购物车/订单接口返回 401 | 统一所有服务使用 Redis DB 0 |
| 3 | CategoryServiceImpl / BannerServiceImpl 缓存操作未加 try-catch | Redis 异常时导致接口报错 | 添加异常捕获，缓存失败降级查库 |
